---

## WORK FROM HOME SUPPORT

Work From Home (WFH) must be implemented within Attendance Management.

Do not create a separate WFH module.

WFH should be managed through:

* Employee Work Mode
* Attendance Policies
* Attendance Capture
* Workflow Engine

---

# Employee Work Modes

Integrate with Employee Master.

Supported values:

* Onsite
* Work From Home
* Hybrid
* Remote

Fields already exist in Employee Master:

* work_mode
* primary_branch

Hybrid employees may optionally define:

* Office Days
* WFH Days

Example:

Monday → Office

Tuesday → WFH

Wednesday → Office

Thursday → WFH

Friday → Office

---

# Attendance Check-In Screen

Enhance Check-In flow.

Before check-in, display:

Where are you working today?

Options:

* Office
* Work From Home

For Hybrid employees:

Preselect based on configured work schedule.

Allow override if policy permits.

Capture:

* Attendance Location Type
* Check-In Source
* Employee Work Mode

Store separately.

Example:

Employee Work Mode: Hybrid

Attendance Location Type: Work From Home

---

# Attendance Location Types

Values:

* Office
* Work From Home
* Client Site
* Remote

Store in attendance_records.

Do not derive from employee work mode.

Employees may work from different locations on different days.

---

# Check-In Rules

## Office Check-In

Require:

* Geofence Validation (Optional)
* Branch Validation
* QR Validation (Optional)
* Biometric Validation (Future)

Capture:

* Latitude
* Longitude
* IP Address

---

## Work From Home Check-In

Require:

* Web Check-In
* Mobile Check-In

Capture:

* IP Address
* Device Information
* Optional Location Coordinates

Do not enforce geofencing.

Future support:

* Activity Tracking
* Screen Monitoring
* Time Tracking

Display:

Coming Soon

Do not implement now.

---

# Hybrid Work Validation

If employee work mode is Hybrid:

Validate expected location.

Example:

Tuesday → WFH

If employee selects Office:

Show warning:

You are scheduled to work from home today.

Allow override if configured.

Policy setting:

Allow Schedule Override

Values:

* Yes
* No
