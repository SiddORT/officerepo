Let's design and implement the Client Portal Skeleton for Office Repo.

Context:

Office Repo is a Multi-Tenant SaaS platform.

Architecture:

Client
↓
Dedicated Database
↓
Enabled Modules
↓
Submodules
↓
Screens

The objective of this phase is NOT to build business functionality.

The objective is to create the complete application skeleton that will support all current and future modules.

Implement:

* Dynamic Sidebar
* Module Registry
* Submodule Registry
* Dynamic Routing

* Layout Structure
* Route Guards
* Placeholder Pages

# Database Design

Create module registry tables.

modules

Fields:

* id
* module_code
* module_name
* parent_module_id
* icon
* route
* display_order
* description
* is_active

client_modules

Fields:

* id
* client_id
* module_id
* is_enabled

Rules:

* Parent modules are top-level modules.
* Submodules reference parent_module_id.
* Enablement is controlled through client_modules.

---

# Module Enablement Rules

Only enabled modules should appear.

Rules:

* Parent module visible only if at least one child module is enabled.
* Child modules visible only if enabled.
* Disabling parent module disables all child modules.
* Enabling parent module enables all child modules by default.

# Route Protection

Validate:

* JWT Authentication
* Module Enabled
* Client Access

If unauthorized:

Return:

403 Forbidden

Message:

Module not enabled for this client.
