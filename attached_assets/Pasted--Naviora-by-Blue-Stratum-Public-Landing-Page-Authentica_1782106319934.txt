# Authentication Strategy

Use a single users table.

Database table:

users

Fields:

* id

* uuid

* first_name

* last_name

* email

* password_hash

* user_type

* organization_id

* is_active

* last_login_at

* created_at

* updated_at

User Types:

* ADMIN
* ORGANIZATION
* CANDIDATE

Rules:

* organization_id is NULL for ADMIN users.
* organization_id is mandatory for ORGANIZATION users.
* organization_id is mandatory for CANDIDATE users.

---

# Login Rules

Admin users:

* Can authenticate only via:

```text
/admin/login
```

Organization users:

* Can authenticate only via:

```text
/org/login
```

Candidate users:

* Can authenticate only via:

```text
/candidate/login
```

Prevent cross-portal authentication.

Examples:

* Candidate credentials must not work on admin login.
* Admin credentials must not work on organization login.
* Organization credentials must not work on candidate login.

Validate user type on backend and frontend.

Return HTTP 403 when portal and user type do not match.

# JWT Requirements

```json
{
  "user_id": "",
  "user_type": "",
  "organization_id": ""
}
```

Use claims for authorization checks.

Do not trust frontend route protection.

All validation must happen server-side.
