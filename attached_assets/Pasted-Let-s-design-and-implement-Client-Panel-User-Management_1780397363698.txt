Let's design and implement Client Panel → User Management for Office Repo.

Context:

Office Repo is a Multi-Tenant SaaS platform.

Each Client has:

* Dedicated Database
* Dedicated Users
* Dedicated Roles
* Dedicated Activity Logs
* Dedicated Authentication

Technology Stack:

* React
* TailwindCSS
* FastAPI
* SQLAlchemy 2.0
* Alembic
* PostgreSQL
* JWT Authentication
* Clean Architecture
* Repository Pattern
* Service Layer Pattern

This is the first module that will be available after a Client Panel is provisioned.

Permissions will be implemented later as business modules are developed.

---

User Management

# Users

Manage client users.

Fields:

* Employee Code (Future)
* First Name
* Last Name
* Display Name
* Email
* Mobile Number
* Status
* Profile Picture
* Last Login
* Is Active

Actions:

* Create User
* Edit User
* View User
* Activate User
* Deactivate User
* Reset Password
* Unlock User
* Force Logout

---

# Roles

Manage client roles.

Initial Roles:

* Super Admin
* Admin
* Manager
* Employee

Fields:

* Role Name
* Description
* Status
* Is System Role

Actions:

* Create Role
* Edit Role
* Clone Role
* Activate Role
* Deactivate Role

Note:

Role-based permissions will be implemented in a later phase.

For now roles act as user classifications only.
