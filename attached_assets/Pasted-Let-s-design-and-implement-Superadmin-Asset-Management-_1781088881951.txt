Let's design and implement Superadmin → Asset Management Setup for Office Repo.

Context:

Office Repo is a Multi-Tenant SaaS platform.

Before Clients can use Asset Management, Superadmin must define the global asset structure.

This setup will be shared across all clients.

Asset Management will be used by:

* Employees
* Departments
* Branches
* IT Teams
* HR Teams
* Procurement Teams

Future Features:

* Asset Assignment
* Asset Returns
* Asset Transfers
* Asset Maintenance
* Asset Audits
* Asset Disposal
* Software License Management

Design Asset Management Setup consisting of:

1. Asset Categories
2. Asset Sub Categories
3. Asset Masters

Follow Office Repo standards:

* FastAPI
* SQLAlchemy 2.0
* PostgreSQL
* Alembic
* React
* TailwindCSS
* Clean Architecture
* Repository Pattern
* Service Layer Pattern
