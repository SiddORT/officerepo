IMPORTANT ARCHITECTURAL DECISION

Office Repo is a multi-tenant SaaS platform.

However, throughout the application we use the business term:

CLIENT

instead of:

TENANT

A Client is a Tenant.

Do NOT create separate Client and Tenant entities.

There should be only one primary SaaS entity:

Client

Each Client represents:

- One organization
- One isolated database
- One set of users
- One subscription
- One module configuration
- One storage space
- One branding configuration

MULTI TENANT REQUIREMENTS
====================================================

Maintain database-per-client architecture.

Do NOT remove multi-tenant design.
