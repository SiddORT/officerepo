Let's design and implement Settings → Notification Management for Office Repo.

Context:

Office Repo uses:

* React
* TailwindCSS
* FastAPI
* SQLAlchemy 2.0

* JWT Authentication

* Clean Architecture
* Repository Pattern
* Service Layer Pattern
* Multi-Tenant SaaS (Database Per Client)

Notification Management is an organization-level module responsible for configuring all communication channels, templates, event triggers, delivery tracking, and usage monitoring.

The module should support:

# Notification Management

## Channels

### Email

SMTP Configuration:

* SMTP Host
* SMTP Port
* Username
* Password
* Encryption (TLS/SSL)
* Sender Name
* Sender Email
* Reply-To Email

Features:

* Test Connection
* Send Test Email
* Enable/Disable Channel

---

### SMS

Provider Configuration:

* Provider Name
* API Key
* API Secret
* Sender ID
* Country
* Route

Features:

* Test SMS
* Enable/Disable Channel
* Credit Monitoring

Supported Providers (Extensible):

* Twilio
* MSG91
* TextLocal
* AWS SNS
* Custom Provider

---

### WhatsApp Business

Configuration:

* Business Account ID
* Phone Number ID
* Access Token
* Verify Token
* Webhook URL

Features:

* Connection Status
* Template Sync
* Test Message
* Enable/Disable Channel

---

### Push Notifications

Firebase Cloud Messaging (FCM)

Configuration:

* Firebase Project ID
* Sender ID
* Server Key
* Service Account Credentials

* Test Push Notification
* Enable/Disable Channel

Platforms:

* Web
* Android
* iOS
