- Client Management
- Subscriptions
- Invoices
- Payments
- Reports
- Billing
- Analytics

Currencies are configured centrally and then assigned to clients.

====================================================
BASE CURRENCY
====================================================

Only one currency can be Base Currency.

Example:

USD

or

INR

Base Currency is used for:

- Revenue Reporting
- Consolidated Analytics
- Currency Conversions
- Dashboard Reporting

Changing Base Currency requires confirmation.

Track in audit logs.

====================================================
EXCHANGE RATE MANAGEMENT
====================================================

Each currency must support:

Current Rate

Rate Source

Auto Sync Enabled

====================================================
AUTO RATE SYNC
====================================================

Prepare integration architecture for:

Primary Sources:

- ExchangeRate API
- Fixer API
- Open Exchange Rates

Future Sources:

- XE
- CurrencyLayer

Do not hardcode providers.

Create provider abstraction.

Example:

exchange_rate_provider.py

====================================================
MANUAL OVERRIDE
====================================================

Finance/Admin users must be able to:

Override exchange rates manually.

Example:

Forex API:
1 USD = 83.24 INR

Internal Accounting:
1 USD = 83.00 INR

Store:

is_manual_override = true

Track all overrides.
